# 파일 태그 기반 분류기 (C#)

이 프로젝트는 업로드된 파일의 파일명에 있는 [Tag] 텍스트를 바탕으로 파일을 적절한 폴더로 이동(또는 복사)합니다.

주요 기능
- 파일명에서 [Tag] 추출 (예: "[Setup]dodo.txt" → "Setup")
- 태그별 폴더 매핑 가능 (tagMap)
- 대상 폴더가 없으면 생성
- 파일명/폴더명 안전하게 정리(파일명에 문제가 되는 문자 대체)
- 이름 충돌 시 "이름 (1).ext" 형태로 고유화
- 이동(move) 또는 복사(copy) 옵션 제공

사용 예
- 콘솔: dotnet run -- <sourceFile> <destRoot> [move|copy]

로컬에서 main 브랜치 생성 및 푸시 (예시)
1. 로컬에서 작업 디렉터리를 준비:
   - git init (또는 기존 리포지토리에서 작업)
2. main 브랜치 생성 및 체크아웃:
   - git checkout -b main
3. 파일 추가, 커밋:
   - git add .
   - git commit -m "Add TagRouter C# implementation"
4. 원격 리포지토리가 이미 설정되어 있다면 main 브랜치 푸시:
   - git push -u origin main

원격 리포지토리가 비어있거나 새로 생성된 경우, 위 push 명령으로 main 브랜치가 원격에 생성됩니다.

테스트
- xUnit 또는 NUnit을 사용해 간단한 유닛 테스트 작성 권장
- 예: 임시 디렉터리에 파일을 만든 뒤 RouteFile을 호출하고 결과 파일 존재 여부 및 폴더명을 검증

주의사항
- 서비스에서 파일 업로드를 처리할 경우, 업로드가 끝난 후 RouteFile을 호출하세요. 스트리밍 중이라면 스트리밍을 최종 대상 경로로 직접 쓰는게 더 효율적입니다.
- 파일/디렉터리 권한을 확인하세요 (서비스 계정이 쓰기 가능한지).
- 대량 처리 시 비동기 작업 또는 큐를 고려하세요.
